Software Requirements

Introduction

This software enables users to make searches through twitter. Users can make research about the topics which they are willing. Software will give us the topic and related keywords usage frequency change through the time. For instance, the topic you are searching  is COVID. The software will search COVID as a main topic and other keywords such as lockdown, vaxxine, trump. The software would show the results at the time dependent graph which you could see the usage frequency depends on the time.

Project Scope

        Project is prepared as a SWE573 class term project. The search results will not be shown to the public. The main usage will be Covid related posts. There could be other topic items that could be added as updates to the software. The software will be worked as a web application. Project is mainly for learning purposes, not for end users.

System Overview

        There will be a web page, where users will login and make their searches. The previous searches will be stored in the system and users  may investigate the results whenever they want. Twitter API’s will be used for searching through the tweets. In order to get more accurate results entity search will be implemented. There will be specific entities for each topic. And they will be added to Twitter API as a keyword filter. Entities will be bound to the tagme site with an API. So it will be changed every topic you enter.

The twitter API has some limits, so the related topic streamed for days to make a reliable database. The related search will be made in these saved tweets.

Use Case Scenario

        Basically the user could able to achieve following functions, Login, Topic Selection & Search, View Search Results, Change Chart Type,  Save the Results, View Previous Results.You can investigate the use case scenario chart (Figure-1),

The main process flow is assumed as follows (Figure-2). User enters the webpage, then login into his account. Then enters the specific topic which is looking for. Then related entity recommendations appear. These entities will make the research more clear and more relevant tweets will appear.  If the user wants, then changes the entities (keywords). After that search will be conducted. Related results will be shown as graphical distribution. These graphs could be changeable with user desires. Then these results will save the search history of the user.
